# プロジェクトルール(テンプレート)

このリポジトリは、業種を問わず使える「個人 / 小規模ビジネス向けランディングサイト + ブログ + チャットボット」テンプレートです。
Claude Code は、ユーザーから「このテンプレで作って」と依頼されたとき、以下のルールに従って動いてください。

---

## 基本ルール

- 返答は**日本語**で行う
- デザインは**モダンで洗練されたもの**にする
- **レスポンシブ対応**(スマホでも見やすい)を必ず行う
- コードのコメントは**日本語**で書く

---

## 動き方(必読)

### Step 1. ヒアリング

最初に **`HEARING_QUESTIONS.md`** をユーザーに提示して、回答を集める。
全項目に答えてもらうのが理想だが、最低限 0〜2(基本情報・トーン・Hero) があれば着手できる。

### Step 2. 設定ファイルを更新

`lib/config.ts` を編集:
- `SITE_NAME` … サイト名
- `SITE_DESCRIPTION` … 一言説明
- `ENABLE_CHATBOT` … チャットボットを表示するか
- `ENABLE_BLOG` … ブログ機能を使うか

### Step 3. コピーとデータを流し込む

| 編集対象 | 反映先 |
|---|---|
| Hero コピー | `components/Hero.tsx` |
| About 文章 | `components/About.tsx` |
| サービス / プラン | `lib/courses.ts` |
| 実績 / 作品 | `lib/works.ts` |
| FAQ | `lib/chatbot/faq.ts`(FAQセクションとチャットボット両方に反映) |
| ブログ記事 | `lib/blog.ts` |
| SNS / フッター | `components/Footer.tsx` |

### Step 4. 不要セクションを外す

ユーザーが「Works いらない」と言ったら、`app/page.tsx` から対応コンポーネントの import と JSX を削除する。

### Step 5. 確認

`npm run dev` で起動し、http://localhost:3000 で各セクションが意図通りに表示されているか確認する。
スマホ表示(375px)・タブレット(768px)・PC(1280px)の崩れをチェック。

### Step 6. デプロイ(任意)

GitHub → Vercel に import するだけで公開できる。
独自ドメインがあれば Vercel の Domains 設定で接続する。

## 構造の理解

詳しい設計図は `TEMPLATE_GUIDE.md` を参照。
コンポーネント・データファイル・配色・タイポグラフィの責務がまとまっている。

---

## 守ってほしいこと

- 既存のデザイントーン(ミニマル + ポップ)を勝手に大幅変更しない
- カラーパレットは `app/globals.css` の CSS 変数を編集することで一元管理する
- 新規セクションを追加するときは `SectionHeading` を使う(番号体系を維持)
- アニメーションは `framer-motion` + `cubic-bezier(0.22, 1, 0.36, 1)` で統一する
